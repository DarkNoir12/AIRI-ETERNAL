## Safest Sync Approach

### Phase 1: Preparation (zero risk)
1. **Add upstream remote** — `git remote add upstream https://github.com/dasilva333/airi.git`
2. **Fetch upstream** — `git fetch upstream`
3. **Backup current state** — `git branch backup/pre-merge` (safety net)

### Phase 2: Merge (conflicts will surface, we resolve them)
4. **Merge upstream** — `git merge upstream/main`
   - Git will auto-merge most files
   - Conflicts will appear in the 10 overlapping files above
   - We resolve each conflict manually, keeping **both** upstream improvements AND our OmniVoice additions

### Phase 3: Verification
5. **Run typecheck** — `pnpm typecheck` to catch any integration issues
6. **Run lint** — `pnpm lint:fix` to fix formatting
7. **Test OmniVoice** — verify the speech provider still works after merge
8. **Push** — `git push origin main`

### Why merge over rebase?
- **Merge** preserves both histories side-by-side. If something breaks, we can see exactly what upstream changed vs what we changed.
- **Rebase** rewrites our commits on top of upstream, which makes it harder to undo if a conflict resolution goes wrong.
- For a first sync with 33 upstream commits, merge is safer.

- **~10 conflict files** to resolve manually
- Most will be straightforward (i18n files = just add our keys to their updated structure)
- The hardest will be `providers.ts`, `speech.ts`, and `Stage.vue` — need to carefully merge our OmniVoice blocks into their refactored code
